================================================================================
                      CODEXION - ROADMAP DE REALISATION
================================================================================

Le projet est une variante du probleme des Philosophes Dinants avec :
  - 3 etats au lieu de 2 (compile / debug / refactor)
  - Cooldown sur les ressources (dongles)
  - Deux politiques de scheduling configurables (fifo / edf)
  - Une priority queue (heap) custom obligatoire
  - Un thread monitor dedie a la detection du burnout (precision <10ms)
  - Garantie de liveness (pas de famine en edf)

ANNEXE - PIEGES CLASSIQUES A NE PAS OUBLIER
================================================================================

* Toujours wrapper cond_wait dans un while (!condition), jamais un if
* cond_wait libere et reprend le mutex, ne PAS unlock avant
* Le sujet n autorise QUE pthread_cond_broadcast (pas cond_signal),
  donc tout reveil sera collectif -> les coders doivent re-tester leur
  condition
* La precision du sleep est critique : usleep n est pas garanti, faire
  une boucle avec verification du temps reel
* Initialiser last_compile_start au debut de la simulation, pas a 0,
  sinon tous les coders seront immediatement en burnout
* Liberer les dongles en sens inverse de l acquisition pour eviter
  les inversions de priorite dans la file d attente
* Le monitor doit lire last_compile_start sous mutex (sinon torn read
  sur 64 bits possible selon archi)
* Pour 1 seul coder : il y a 1 seul dongle sur la table (pas 2),
  donc le coder ne peut jamais compiler -> il finira par burnout.
  C est le comportement attendu, ne pas le forcer a un cas special.
